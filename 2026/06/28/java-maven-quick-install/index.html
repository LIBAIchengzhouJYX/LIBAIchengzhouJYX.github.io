前置准备：PowerShell 执行策略

在 Windows 上执行 PowerShell 脚本之前，需要确保执行策略允许脚本运行。

Set-ExecutionPolicy RemoteSigned -Scope CurrentUser

这是一个 PowerShell 命令，用于设置脚本执行策略。拆开来看：

各部分含义

部分 说明
Set-ExecutionPolicy PowerShell 的 cmdlet，用于修改执行策略
RemoteSigned 策略值：从网络下载的脚本必须经过数字签名才能运行，本地创建的脚本不受限制
-Scope CurrentUser 作用域：该设置仅影响当前用户，不需要管理员权限

RemoteSigned 具体行为

  • 本地脚本（自己写的 .ps1 文件）：可以直接运行，不需要签名
  • 从网络下载的脚本（如浏览器下载、Invoke-WebRequest 获取的）：必须由受信任的发布者进行数字签名，否则会被阻止

-Scope CurrentUser 的好处

  • 不需要以管理员身份运行 PowerShell
  • 比 LocalMachine（影响所有用户）更安全，范围更小
  • 是 Windows 上最常用的开发者配置

常见执行策略对比

策略 说明
Restricted 默认值，禁止所有脚本运行
RemoteSigned 本地脚本可运行，远程脚本需签名（推荐）
AllSigned 所有脚本都必须签名（最严格）
Unrestricted 所有脚本都可运行，远程脚本会有警告
Bypass 无任何限制，无警告
