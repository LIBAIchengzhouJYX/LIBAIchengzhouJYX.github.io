一、项目概述

1.1 什么是 HexStrike AI

HexStrike AI 是一个双脚本 MCP（Model Context Protocol）架构的 AI 驱动渗透测试框架，版本 v6.0。它将 100+ 安全工具封装为 AI 可调用的标准化接口，专为 Bug Bounty / CTF / Red Team / 安全研究设计。

1.2 核心架构

1.3 关键文件清单

文件 部署位置 功能
hexstrike_server.py Linux 服务器 /home/***/***/ Flask API 服务端，管理工具执行、错误恢复、AI 决策引擎
hexstrike_mcp.py Windows 客户端（本地） MCP 协议客户端，通过 FastMCP 向 AI 暴露 100+ 工具接口
setup_mcp_tools.py Windows 客户端 SSH 到服务器，自动补充安装缺失工具（XSStrike / ZAP 等）
start_hexstrike.py Windows 客户端 SSH 到服务器，远程启动 hexstrike 服务进程
check_hexstrike.py Windows 客户端 检查服务器端口（22 / 8888）连通性
copy_mcp.py Windows 客户端 从开发目录同步最新代码到工作目录
run_ssh.py Windows 客户端 安全绕过 safe-rm 后执行 start_hexstrike.py

1.4 源码目录与工作目录

目录 说明
<DEV_DIR> 开发源码目录 — 代码编写和修改的地方
<WORK_DIR> 工作执行目录 — 拷贝后实际运行的地方

每次修改源码后，需执行 python copy_mcp.py 同步到工作目录。

2.3 PowerShell 执行策略设置

各策略说明：

策略 说明

AllSigned 所有脚本都必须签名
Unrestricted 全部可运行，远程会警告
Bypass 无任何限制

4.5 远程启动服务器（可选）

如果服务器端服务未启动，可以通过 SSH 远程启动：

start_hexstrike.py 的 SSH 执行流程：

步骤 操作 SSH 命令
1 检查已有进程 ps aux | grep -i hexstrike | grep -v grep
2 检查端口占用 ss -tlnp | grep 8888
3 查找服务脚本 find /home /opt /root -name 'hexstrike_server.py'
4 检查 Python python3 --version
5 启动服务 cd /home/***/***/ && nohup python3 hexstrike_server.py --host 0.0.0.0 --port 8888 > /tmp/hexstrike.log 2>&1 &
6 验证 curl http://127.0.0.1:8888/health
