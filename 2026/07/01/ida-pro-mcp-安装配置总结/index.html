三、遇到的错误与解决方案

错误 1：GitHub 不可达

现象：

根因：当前网络环境无法访问 GitHub（Connection was reset）。

解决：将 GitHub URL 替换为 Gitee 镜像：

错误 2：PyPI 不可达（SSL EOF）

现象：

根因：即使 Gitee 克隆成功，pip 构建时仍需从 PyPI 下载 setuptools 等构建依赖，但当前网络环境无法直连 https://pypi.org。

解决：添加 -i https://pypi.tuna.tsinghua.edu.cn/simple 参数，使用清华 PyPI 镜像。

注意：Gitee 镜像版本为 v1.3.0，功能上与 GitHub 最新版一致（核心 MCP 桥和 22 个工具均完整），唯一差异是缺少 --config 和 --ida-rpc 等较新加的命令行选项。

错误 3：--config 参数不存在

现象：

根因：当前安装的 v1.3.0（Gitee 镜像）不支持 --config。这个参数是在更高版本才加入的。

替代方案：从 vscode.mcp.config.json（Trae 已提前写入）和 server.py 源码中找到两种 MCP 配置模式：

HTTP 模式（更简洁）：

STDIO 模式（更稳定）：

v1.3.0 的 server.py 只支持 STDIO 模式（mcp.run(transport="stdio")），不支持 HTTP 模式，因此我们选用 STDIO 模式。

错误 4：--install 不认识 Claude Code CLI 和 Trae

现象：ida-pro-mcp --install 只检查了 4 种客户端的配置目录：

  • Cline → <APPDATA>/Code/User/globalStorage/saoudrizwan.claude-dev/settings/cline_mcp_settings.json
  • Roo Code → <APPDATA>/Code/User/globalStorage/rooveterinaryinc.roo-cline/settings/mcp_settings.json
  • Claude Desktop → <APPDATA>/Claude/claude_desktop_config.json
  • Cursor → ~/.cursor/mcp.json

根因：server.py:install_mcp_servers() 只内置了这 4 种客户端的配置路径，没有 Claude Code CLI 也没有 Trae（火星码）。

解决：手动写入 MCP 配置文件。

客户端 配置文件路径 动作
Claude Code CLI %USERPROFILE%\.mcp.json 新建
Trae（火星码） %USERPROFILE%\.marscode\vscode.mcp.config.json 已存在（改写）

注意：Claude Code CLI 的 settings.json 不支持 mcpServers 字段。正确的配置入口是 ~/.mcp.json（用户级）或 .mcp.json（项目级）。

错误 5：试图在 settings.json 中添加 mcpServers

现象：编辑 ~/.claude/settings.json 添加 mcpServers 字段时报错：

根因：Claude Code 的 settings.json 严格校验 schema，mcpServers 不在其字段列表中。MCP 服务器配置使用独立的 .mcp.json 文件。

解决：将 MCP 服务器配置写入 ~/.mcp.json 而非 settings.json。

四、最终配置清单

4.2 Trae（火星码）— %USERPROFILE%\.marscode\vscode.mcp.config.json

已自动安装，无需手动操作。

五、架构全链路

关键流程：

  1. Trae / Claude Code 启动时读取 MCP 配置
  2. 自动 spawn python.exe server.py 子进程
  3. server.py 初始化 MCP 握手（initialize → notifications/initialized）
  4. 用户发出逆向分析请求
  5. Claude Code / Trae 发 tools/call → server.py 通过 stdin 接收
  6. server.py 转发 JSON-RPC 请求 → IDA 插件 localhost:13337/mcp
  7. IDA 插件执行操作，返回结果
  8. server.py 通过 stdout 返回结果给客户端

八、使用流程

推荐的 Prompt 模板（来自官方）

Your task is to analyze a crackme in IDA Pro. You can use the MCP tools to retrieve information. In general use the following strategy:

  • Inspect the decompilation and add comments with your findings
  • Rename variables to more sensible names
  • Change the variable and argument types if necessary (especially pointer and array types)
  • Change function names to be more descriptive
  • If more details are necessary, disassemble the function and add comments with your findings
  • NEVER convert number bases yourself. Use the convert_number MCP tool if needed!
  • Do not attempt brute forcing, derive any solutions purely from the disassembly and simple python scripts
  • Create a report.md with your findings and steps taken at the end
